The Laminar debugger records an agent run, lets you inspect every LLM call and tool use in a transcript, and replays the run from any point with your fix applied. On replay it serves cached LLM responses from the recorded trace up to a boundary you pick, then runs live past it, so you only pay for the calls that actually changed.

<Note>
**The debugger is built for AI coding agents.** Add the [Laminar skill](https://github.com/lmnr-ai/lmnr-skills) to Claude Code, Cursor, or Codex and the agent drives the whole record/inspect/replay loop on its own. A human can run the same loop by hand: every step is a CLI command or an environment variable.
</Note>

## Why it exists

Long-running agents fail slowly. A run takes minutes, the bad call is three-quarters of the way through, and every fix means waiting for the full run again. The feedback loop is too slow to iterate on.

The debugger collapses that loop. Record once, then replay from just before the call you're fixing. The recorded calls before your boundary come back from cache instantly; only the calls after it run live.

This matters most for an AI coding agent fixing your agent. The agent can run dozens of replay iterations while you watch, each one cheap and fast, without re-running the expensive prefix of the trace every time.

## The two primitives

- **Sessions** group every debug run from one investigation. All the traces appear together in the Laminar UI so you (or your coding agent) can compare attempts side by side and follow the notes attached to each run.
- **Replay** re-runs the agent against a recorded trace. The debugger serves cached LLM responses up to the boundary you choose and runs everything past it live. See [how caching works](/debugger/caching) for the mechanism.

## Deploy with Docker Compose

<Steps>
<Step title="Clone the repository">

```bash
git clone https://github.com/lmnr-ai/lmnr
cd lmnr
```
</Step>
<Step title="Start the stack">

```bash
docker compose up -d
```
</Step>
<Step title="Access the dashboard">

Open `http://localhost:5667/`.
</Step>
<Step title="Create a project and API key">

Complete onboarding, then create a project API key in project settings. Copy itâ€”you will not be able to view it again. Add access controls if needed (see below).
</Step>
<Step title="Point the SDK at your instance">

The backend listens on port `8000` (HTTP) and `8001` (gRPC).

<Tabs>
<Tab title="JavaScript/Typescript">

```ts
import { Laminar } from '@lmnr-ai/lmnr';

Laminar.initialize({
  projectApiKey: "<YOUR_PROJECT_API_KEY>",
  baseUrl: "http://localhost",
  httpPort: 8000,
  grpcPort: 8001,
  instrumentModules: {
    // your libraries to instrument
  },
});
```
</Tab>
<Tab title="Python">

```python
from lmnr import Laminar

Laminar.initialize(
    project_api_key="<YOUR_PROJECT_API_KEY>",
    base_url="http://localhost",
    http_port=8000,
    grpc_port=8001,
)
```
</Tab>
</Tabs>
</Step>
</Steps>

## Access control

Enable GitHub authentication by configuring the frontend with:

```
AUTH_GITHUB_ID=your_github_oauth_app_id
AUTH_GITHUB_SECRET=your_github_oauth_app_secret
```

Default behavior allows any GitHub user. To restrict access, add `allowed-emails.json` in the project root:

```json
{
  "emails": [
    "user1@example.com",
    "user2@example.com"
  ]
}
```

For more deployment options and contributions, see the README in the GitHub repository.

## Slack

The Slack integration lets you receive real-time notifications in your Slack channels when [signal](/signals/introduction) events occur. Once connected, notifications work across all projects in the workspace.

<Note>
  Slack integration is available on paid plans only.
</Note>

### Connect Slack

1. Go to your **Workspace** page and select **Integrations** from the sidebar.
2. Find the **Slack** card and click **Connect to Slack**.
3. You will be redirected to Slack's authorization page. Select the Slack workspace you want to connect and click **Allow**.
4. After authorization, you will be redirected back to Laminar. The Slack card will show a **Connected** badge with your Slack workspace name.

The Laminar Slack app requests permission to post messages to public channels in your Slack workspace. It does not access private channels.

### Remove Slack integration

1. Go to your **Workspace** page and select **Integrations** from the sidebar.
2. On the connected Slack card, click the dropdown menu (three dots).
3. Select **Remove**.
4. Confirm the removal in the dialog.

<Warning>
  Removing the integration deletes all alert subscriptions connected to that Slack workspace. This action cannot be undone.
</Warning>

### What you can do with Slack connected

- **[Alerts](/signals/alerts)** - Create per-project alerts that send Slack notifications to a channel of your choice whenever a signal event is detected.
- **[Laminar Agent](/platform/laminar-agent)** - Mention `@Laminar` in a connected channel to ask about your traces, costs, signals, and more in plain language. The agent queries your project's data and replies in the thread.

### Use the agent in Slack

Once Slack is connected, mention `@Laminar` in any channel and the agent replies in that thread. Ask it the same kinds of questions you would ask it in the [CLI](/platform/cli) or an [MCP client](/platform/mcp):

> @Laminar what failed in the most recent run, and why?

The agent runs read-only SQL and reads trace context to ground its answer, then posts the reply back into the thread. Follow-up mentions in the same thread continue the conversation.

#### Connect a channel to a project

The agent answers about one Laminar project per channel, so each channel is connected to a project before the agent can query its data:

1. Mention `@Laminar` in a channel that is not yet connected.
2. The agent replies with a project picker listing the projects available to your workspace. Choose one.
3. The channel is now connected to that project. Mention the agent again and it answers from that project's data.

A workspace admin can also connect a channel to a project from **Workspace → Integrations**. Each channel maps to a single project; pick the project whose traces the people in that channel care about.

**Table views** save a table's filters, search, sort, and column layout as a named preset you can reopen with one click and share with the rest of the project.

<Frame caption="The view picker on the traces table, with three saved views.">
  <img src="/images/platform/table-views/picker.png" alt="Table views picker open above the traces table" />
</Frame>

A typical use: filter Traces to `status = error` in the last hour, save it as **Errors in last hour**, and any teammate hitting the Traces tab can pick the same preset from the dropdown instead of rebuilding the filter chain.

Each view is scoped to one project and one table (Traces, Spans, Sessions, Evaluations, Datasets, and the others), so saving a view on Traces never clutters the dropdown on Spans.

## Save a view

Set up the table the way you want it. The picker shows an amber dot when the current state has drifted from any saved view; open it and the **Unsaved changes** section appears:

- **Save changes** writes back to the active view.
- **Save as new view** prompts for a name.
- **Discard changes** reverts to the active view (or to `Default view` if none is loaded).

<Frame caption="The Unsaved changes section appears when the table state has drifted from the active view.">
  <img src="/images/platform/table-views/unsaved.png" alt="Unsaved changes section in the views dropdown" />
</Frame>

## Switch, rename, delete

Picking a view from the dropdown applies its filters, search, sort, and columns and writes `?v=<view-id>` to the URL, so you can paste the link into Slack and your teammate lands on the same table state. Hover any row in the dropdown for inline rename and delete.

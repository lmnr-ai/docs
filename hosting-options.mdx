Laminar is open-source, so you can run it on Laminar Cloud (managed) or self-host the full stack yourself. Self-host with [Docker Compose](#self-hosted-docker-compose) for local and single-node setups, or the [Kubernetes Helm chart](#self-hosted-on-kubernetes-helm) for production.

## Managed (Laminar Cloud)

1. Sign up at [laminar.sh](https://laminar.sh).
2. Create a project.
3. Copy your project API key from project settings.
4. Set `LMNR_PROJECT_API_KEY` in your environment.

Next: start instrumenting your app by following the tracing entry point: [Tracing](/tracing/introduction).

## Self-hosted (Docker Compose)

Run the full Laminar stack locally with Docker Compose:

```bash
git clone https://github.com/lmnr-ai/lmnr
cd lmnr
docker compose up -d
```

The dashboard is available at `http://localhost:5667/`. Create a project through the onboarding flow, then grab your API key from project settings.

### Access control

By default, any user can access your self-hosted instance. To restrict access to specific GitHub users, enable GitHub OAuth:

```bash
AUTH_GITHUB_ID=your_github_oauth_app_id
AUTH_GITHUB_SECRET=your_github_oauth_app_secret
```

Then create an `allowed-emails.json` file in your project root:

```json
{
  "emails": [
    "user1@example.com",
    "user2@example.com"
  ]
}
```

## Self-hosted on Kubernetes (Helm)

For production deployments, run Laminar on Kubernetes with the [Laminar Helm chart](https://github.com/lmnr-ai/lmnr-helm). The chart deploys the full stack: frontend, app server, PostgreSQL, ClickHouse, Redis, RabbitMQ, and Quickwit. EKS and GKE are the recommended clusters.

### Prerequisites

- A Kubernetes cluster (EKS or GKE recommended)
- Helm 3.x
- An S3-compatible bucket for trace storage (AWS S3 or GCS with HMAC credentials)
- **AWS**: the [AWS Load Balancer Controller](https://docs.aws.amazon.com/eks/latest/userguide/aws-load-balancer-controller.html) and [EBS CSI Driver](https://docs.aws.amazon.com/eks/latest/userguide/ebs-csi.html)
- **GCP**: the built-in GCE Ingress controller and GCE Persistent Disk CSI Driver

### Install

Add the Helm repository and clone the chart repo to get the `laminar.yaml` values file you'll edit:

```bash
helm repo add laminar https://lmnr-ai.github.io/lmnr-helm
helm repo update

git clone https://github.com/lmnr-ai/lmnr-helm.git
cd lmnr-helm
```

Edit `laminar.yaml` and replace every placeholder value (`<region>`, `<bucket-name>`, and so on) with your real cloud provider, credentials, buckets, and availability zones. At minimum, set:

- `global.cloudProvider` to `aws` or `gcp`.
- Cloud credentials and the S3 bucket for trace storage.
- `AEAD_SECRET_KEY`, generated with `openssl rand -hex 32`. This encrypts project and model API keys.
- ClickHouse and Quickwit S3 bucket endpoints and regions.
- Availability zones (required for AWS EBS volumes).

<Warning>
Angle-bracket placeholders like `<region>` produce invalid XML in the ClickHouse config and cause `CrashLoopBackOff` errors if left unchanged. Replace all of them before installing.
</Warning>

Install the chart, then point the frontend at its load balancer once the address is provisioned:

```bash
# 1. Install
helm upgrade -i laminar laminar/laminar -f laminar.yaml

# 2. Get the frontend ALB URL (wait 1-2 minutes for provisioning)
ALB_URL=$(kubectl get ingress laminar-frontend-ingress -o jsonpath='{.status.loadBalancer.ingress[0].hostname}')

# 3. Configure frontend URLs
helm upgrade -i laminar laminar/laminar -f laminar.yaml \
  --set frontend.env.nextauthUrl="http://$ALB_URL" \
  --set frontend.env.nextPublicUrl="http://$ALB_URL"
```

See the [QUICKSTART](https://github.com/lmnr-ai/lmnr-helm/blob/main/QUICKSTART.md) for step-by-step installation and the [CONFIGURATION reference](https://github.com/lmnr-ai/lmnr-helm/blob/main/CONFIGURATION.md) for OAuth login, TLS, custom domains, and external secret management.

### Point the SDK to your cluster

Get the app server's load balancer address and use it as the SDK base URL:

```bash
LMNR_BASE_URL=$(kubectl get svc laminar-app-server-load-balancer -o jsonpath='{.status.loadBalancer.ingress[0].hostname}') && echo $LMNR_BASE_URL
```

### Slack notifications

Alerts and Slack notifications are an enterprise feature. Set an enterprise license key in `secrets.data` to connect your self-hosted instance to the Laminar Slack app:

```yaml
secrets:
  data:
    LMNR_LICENSE_KEY: "your-enterprise-license-key"
```

To get a license key, contact [founders@lmnr.ai](mailto:founders@lmnr.ai).

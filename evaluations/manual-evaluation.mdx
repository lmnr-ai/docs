`evaluate()` is the right entry point for 95% of cases. For the other 5% (wiring evaluations into an existing pipeline, streaming datapoints from a long-running job, mixing evaluation bookkeeping with production traffic) use the lower-level `LaminarClient.evals` API.

## When to use this

Reach for the manual API when you need to:

- Create an evaluation now and append datapoints to it over hours or days.
- Run the executor and evaluators as part of an existing system rather than a dedicated eval script.
- Produce Laminar datapoints from a production trace without re-running the call.
- Attach custom span structure around the evaluation that `evaluate()` doesn't let you shape.

If none of those apply, use [`evaluate()`](/evaluations/quickstart).

## How it works

Three calls, in order:

1. **`evals.create`** opens a new evaluation. Returns an `evalId`.
2. **`evals.createDatapoint`** records one datapoint. Pass `traceId = Laminar.getTraceId()` from inside a span so the datapoint links to the trace Laminar is currently producing.
3. **`evals.updateDatapoint`** writes scores and the executor output back once the executor and evaluators run.

The same wrapping that `evaluate()` produces (`EVALUATION` root span, `EXECUTOR`, `EVALUATOR`) is yours to produce via `observe()` with the right `spanType`. Laminar joins the spans and the datapoint by `traceId`.

## Setup

<CodeGroup>
```typescript TypeScript
import { Laminar, LaminarClient, observe } from '@lmnr-ai/lmnr';
import { OpenAI } from 'openai';

Laminar.initialize({
  projectApiKey: process.env.LMNR_PROJECT_API_KEY,
  instrumentModules: { OpenAI },
});

const client = new LaminarClient({
  projectApiKey: process.env.LMNR_PROJECT_API_KEY,
});

const openai = new OpenAI();
```

```python Python
import asyncio
from lmnr import Laminar, LaminarClient, observe
from openai import AsyncOpenAI

Laminar.initialize(project_api_key=os.environ["LMNR_PROJECT_API_KEY"])

client = LaminarClient(project_api_key=os.environ["LMNR_PROJECT_API_KEY"])
openai_client = AsyncOpenAI()
```
</CodeGroup>

## Build executor and evaluator spans

Wrap your executor and each evaluator with `observe()`, passing the `EXECUTOR` and `EVALUATOR` span types. Without these span types, the trace still records, but the evaluation UI won't recognize the spans for drill-down.

<CodeGroup>
```typescript TypeScript
const runExecutor = async (testCase: { data: { country: string } }) =>
  observe(
    { name: 'executor', spanType: 'EXECUTOR', input: testCase.data },
    async () => {
      const response = await openai.chat.completions.create({
        model: 'gpt-5-mini',
        messages: [
          {
            role: 'user',
            content:
              `What is the capital of ${testCase.data.country}? ` +
              'Answer only with the capital, no other text.',
          },
        ],
      });
      return response.choices[0].message.content ?? '';
    },
  );

const accuracy = async (output: string, target: string) =>
  observe(
    { name: 'accuracy', spanType: 'EVALUATOR', input: { output, target } },
    async () => (output.includes(target) ? 1 : 0),
  );
```

```python Python
@observe(name="executor", span_type="EXECUTOR")
async def run_executor(test_case):
    response = await openai_client.chat.completions.create(
        model="gpt-5-mini",
        messages=[
            {
                "role": "user",
                "content": (
                    f"What is the capital of {test_case['data']['country']}? "
                    "Answer only with the capital, no other text."
                ),
            }
        ],
    )
    return response.choices[0].message.content or ""


@observe(name="accuracy", span_type="EVALUATOR")
async def accuracy(output, target):
    return 1 if target in output else 0
```
</CodeGroup>

## Create the evaluation and datapoints

<CodeGroup>
```typescript TypeScript
const evalId = await client.evals.create({
  name: 'Manual capitals eval',
  groupName: 'capitals-manual',
});

for (let i = 0; i < testData.length; i++) {
  await observe(
    { name: 'evaluation', spanType: 'EVALUATION', input: { testCase: testData[i] } },
    async () => {
      const testCase = testData[i];

      const datapointId = await client.evals.createDatapoint({
        evalId,
        data: testCase.data,
        target: testCase.target,
        index: i,
        traceId: Laminar.getTraceId(),
      });

      const output = await runExecutor(testCase);
      const accuracyScore = await accuracy(output, testCase.target);

      await client.evals.updateDatapoint({
        evalId,
        datapointId,
        scores: { accuracy: accuracyScore },
        executorOutput: { response: output, model: 'gpt-5-mini' },
      });
    },
  );
}

await Laminar.flush();
```

```python Python
eval_id = client.evals.create_evaluation(
    name="Manual capitals eval",
    group_name="capitals-manual",
)


@observe(name="evaluation", span_type="EVALUATION")
async def run_one(eval_id, i, test_case):
    datapoint_id = client.evals.create_datapoint(
        eval_id=eval_id,
        data=test_case["data"],
        target=test_case["target"],
        index=i,
        trace_id=Laminar.get_trace_id(),
    )

    output = await run_executor(test_case)
    accuracy_score = await accuracy(output, test_case["target"])

    client.evals.update_datapoint(
        eval_id=eval_id,
        datapoint_id=datapoint_id,
        scores={"accuracy": accuracy_score},
        executor_output={"response": output, "model": "gpt-5-mini"},
    )


async def main():
    for i, test_case in enumerate(test_data):
        await run_one(eval_id, i, test_case)
    Laminar.flush()
```
</CodeGroup>

<Note>
`createDatapoint` / `create_datapoint` must be called from *inside* the `EVALUATION` span context so `Laminar.getTraceId()` returns the trace this datapoint is being produced on. Call it before the executor runs so the resulting trace is labelled as evaluation-type up front.
</Note>

## Saving datapoints in bulk

If you already have outputs and scores and just want to register them as an evaluation (no live executor), use `saveDatapoints` / `save_datapoints`:

<CodeGroup>
```typescript TypeScript
const evalId = await client.evals.create({
  name: 'Backfilled evaluation',
  groupName: 'capitals',
});

await client.evals.saveDatapoints({
  evalId,
  datapoints: rows.map((row, i) => ({
    index: i,
    data: row.data,
    target: row.target,
    executorOutput: row.output,
    scores: row.scores,
  })),
});
```

```python Python
eval_id = client.evals.create_evaluation(
    name="Backfilled evaluation",
    group_name="capitals",
)

client.evals.save_datapoints(
    eval_id=eval_id,
    datapoints=[
        {
            "index": i,
            "data": row["data"],
            "target": row["target"],
            "executor_output": row["output"],
            "scores": row["scores"],
        }
        for i, row in enumerate(rows)
    ],
)
```
</CodeGroup>

`saveDatapoints` doesn't attach traces. Use it when you've already run the executor elsewhere (production traffic, offline batch, a different orchestrator) and just want the numbers in Laminar.

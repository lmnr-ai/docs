## Overview

The [AI SDK](https://ai-sdk.dev/docs/introduction) is Vercel's library for building LLM features in JavaScript and TypeScript. It emits [OpenTelemetry](https://opentelemetry.io/) spans for `generateText`, `streamText`, `embed`, `embedMany`, tool calls, and multi-step agents when you enable `experimental_telemetry`.

Laminar's SDK is OpenTelemetry-native, so it picks those spans up directly. You get model inputs and outputs, token usage, latencies, tool-call arguments and results, and the structure of multi-step agent runs.

<Note>
The AI SDK's telemetry API is marked `experimental_*`. The shape has been stable across AI SDK v3, v4, v5, and v6, but the prefix may change in a future major release.
</Note>

<Step title="Initialize Laminar">
<Tabs>
<Tab title="Next.js">
Place `Laminar.initialize` in `instrumentation.ts` at the project root. Next.js loads this file once, early in the app lifecycle, before any route code runs.

```typescript instrumentation.ts
export async function register() {
  if (process.env.NEXT_RUNTIME === 'nodejs') {
    const { Laminar } = await import('@lmnr-ai/lmnr');

    Laminar.initialize({
      projectApiKey: process.env.LMNR_PROJECT_API_KEY,
    });
  }
}
```

The `NEXT_RUNTIME === 'nodejs'` guard is required. Laminar depends on Node.js-only OpenTelemetry APIs and will fail to load in the Edge runtime.

Then tell Next.js to keep `@lmnr-ai/lmnr` out of its bundling pipeline:

```typescript next.config.ts
const nextConfig = {
  serverExternalPackages: ['@lmnr-ai/lmnr'],
};

export default nextConfig;
```

This is required because Laminar uses Node.js-specific OpenTelemetry internals that Next.js's bundler cannot process. See the [Next.js docs on `serverExternalPackages`](https://nextjs.org/docs/app/api-reference/config/next-config-js/serverExternalPackages) for details.

<Note>
If you also use `@vercel/otel` or `@sentry/nextjs`, initialize Laminar **after** those packages register their tracer providers, so Laminar can attach to the same provider.
</Note>
</Tab>
<Tab title="Node.js">
Call `Laminar.initialize` once, at the entry point of your application, before any AI SDK calls run.

```typescript
import { Laminar } from '@lmnr-ai/lmnr';

Laminar.initialize({
  projectApiKey: process.env.LMNR_PROJECT_API_KEY,
});
```
</Tab>
</Tabs>
</Step>

<Step title="Pass the Laminar tracer to your AI SDK calls">

Enable `experimental_telemetry` on every `generateText`, `streamText`, `embed`, `embedMany`, or `generateObject` call, and pass Laminar's tracer:

```typescript {3,8-11}
import { openai } from '@ai-sdk/openai';
import { generateText } from 'ai';
import { getTracer } from '@lmnr-ai/lmnr';

const { text } = await generateText({
  model: openai('gpt-5'),
  prompt: 'What is Laminar flow?',
  experimental_telemetry: {
    isEnabled: true,
    tracer: getTracer(),
  },
});
```

You can add `functionId` and `metadata` to the `experimental_telemetry` object to name the operation and attach searchable attributes:

```typescript
experimental_telemetry: {
  isEnabled: true,
  tracer: getTracer(),
  functionId: 'summarize-ticket',
  metadata: { userId: '42', tier: 'pro' },
},
```
</Step>

## Group multiple calls into one trace

By default, each top-level AI SDK call starts its own trace. When a single request makes several model calls (e.g. plan, then answer), wrap the work in `observe` so the calls share a trace and appear as steps of one operation:

```typescript app/api/chat/route.ts {2, 9-27}
import { openai } from '@ai-sdk/openai';
import { getTracer, observe } from '@lmnr-ai/lmnr';
import { generateText } from 'ai';

export async function POST(req: Request) {
  const { messages } = await req.json();
  const tracer = getTracer();

  const text = await observe(
    { name: 'POST /api/chat' },
    async () => {
      const plan = await generateText({
        model: openai('gpt-5'),
        prompt: `Plan how to answer: ${messages.at(-1).content}`,
        experimental_telemetry: { isEnabled: true, tracer },
      });

      const answer = await generateText({
        model: openai('gpt-5'),
        prompt: `Given plan ${plan.text}, answer the user.`,
        experimental_telemetry: { isEnabled: true, tracer },
      });

      return answer.text;
    },
  );

  return Response.json({ text });
}
```

If your app is already instrumented with `@vercel/otel` or `@sentry/nextjs`, inbound HTTP spans will group the AI SDK calls for you and you can skip `observe`.

See the [`observe` reference](/sdk/observe) for options like `name`, `sessionId`, `userId`, `metadata`, and `tags`.

## Using non-AI-SDK clients alongside the AI SDK in Next.js

If part of your Next.js app calls the OpenAI or Anthropic SDK directly (not through the AI SDK), the Laminar auto-instrumentation that normally runs in `Laminar.initialize` cannot reach those imports, because Next.js loads `instrumentation.ts` in a separate module scope. Call `Laminar.patch` in the file that creates the client:

```typescript lib/openai.ts {2-4}
import OpenAI from 'openai';
import { Laminar } from '@lmnr-ai/lmnr';

Laminar.patch({ OpenAI });

export const openai = new OpenAI();
```

The same pattern applies to `@anthropic-ai/sdk` and other LLM SDKs. Use `Laminar.patch` only in Next.js: in plain Node.js, `Laminar.initialize` handles this automatically.

## Overview

[Claude Agent SDK](https://docs.claude.com/en/api/agent-sdk/overview) (formerly the Claude Code SDK) is Anthropic's framework for running Claude-powered agents from your own code. It handles the agent loop, subagent delegation, tool calls, permission prompts, and the model protocol so you can focus on the task.

Laminar records the whole run as one trace:

- The top-level `query()` / `ClaudeSDKClient` call that drives the turn.
- Every LLM request and response, with prompts, token counts, latency, and cost.
- Every tool call Claude makes (`Read`, `Grep`, `Edit`, `Bash`, your custom tools) with arguments and results.
- **Every subagent as a nested Agent span**, with its own input, output, model calls, and tool calls. The transcript view collapses each subagent to its input and final answer so you read the run as a conversation, not a span tree.

<Step title="Wrap the query function">

Import `query` as `origQuery`, initialize Laminar, then wrap it. Laminar returns a drop-in replacement that traces every turn, subagent, and tool call.

```typescript {2,5,8}
import { query as origQuery } from '@anthropic-ai/claude-agent-sdk';
import { Laminar } from '@lmnr-ai/lmnr';

Laminar.initialize();

const query = Laminar.wrapClaudeAgentQuery(origQuery);

const result = query({
  prompt: 'Review sample_module.py for style, security, and test coverage.',
  options: {
    allowedTools: ['Read', 'Grep', 'Glob', 'Agent'],
    agents: {
      'style-reviewer': {
        description: 'Reviews code for style, readability, and naming.',
        prompt: 'You are a senior engineer focused on code style. Give short, direct feedback.',
        tools: ['Read', 'Grep', 'Glob'],
        model: 'inherit',
      },
      'security-reviewer': {
        description: 'Reviews code for security issues.',
        prompt: 'You are a security engineer. Look for input validation, secrets, and injection issues.',
        tools: ['Read', 'Grep', 'Glob'],
        model: 'inherit',
      },
    },
  },
});

for await (const message of result) {
  console.log(message);
}
```

Include `'Agent'` in `allowedTools` whenever you define subagents: that is the tool Claude calls to delegate to them.

<Step title="Install the SDKs">

Instrumentation loads automatically when both packages are importable. You need `lmnr>=0.7.47` and `claude-agent-sdk>=0.1.0`.


The Python instrumentor uses `lmnr-claude-code-proxy` (already a core dependency of `lmnr`) to inject trace context into each subprocess spawned by the Claude Agent SDK, so parent-child relationships between the outer `query()` and its subagents, tools, and LLM calls are preserved.

<Step title="Initialize Laminar and run your agent">

Call `Laminar.initialize()` once at the entry point. Wrap the top of your agent in `@observe()` so the whole run shares one trace.

```python {4-6,9}
import asyncio

from claude_agent_sdk import AgentDefinition, ClaudeAgentOptions, query
from lmnr import Laminar, observe

Laminar.initialize()


@observe(name="code-review")
async def main():
    options = ClaudeAgentOptions(
        allowed_tools=["Read", "Grep", "Glob", "Agent"],
        agents={
            "style-reviewer": AgentDefinition(
                description="Reviews code for style, readability, and naming.",
                prompt="You are a senior engineer focused on code style. Give short, direct feedback.",
                tools=["Read", "Grep", "Glob"],
                model="inherit",
            ),
            "security-reviewer": AgentDefinition(
                description="Reviews code for security issues.",
                prompt="You are a security engineer. Look for input validation, secrets, and injection issues.",
                tools=["Read", "Grep", "Glob"],
                model="inherit",
            ),
        },
    )

    async for message in query(
        prompt="Review sample_module.py for style, security, and test coverage.",
        options=options,
    ):
        print(message)


if __name__ == "__main__":
    asyncio.run(main())
```

Include `"Agent"` in `allowed_tools` whenever you define subagents: that is the tool Claude calls to delegate to them.

## Track outcomes with Signals

Traces tell you what happened on one run. [Signals](/signals/introduction) turn that into structured outcomes: describe a behavior or failure in natural language (*did the agent finish the punch-list under 150 words, did any subagent skip a file, did a tool call error out*) and Laminar extracts matching events across your history and every new trace. Route them to alerts or datasets.

## Troubleshooting

<AccordionGroup>
  <Accordion title="I see the top-level query span but no subagent spans">
    Make sure `allowedTools` (TS) or `allowed_tools` (Python) includes `'Agent'`. Without it, Claude cannot invoke your subagents at all, so they never run and therefore never appear in the trace. The outer model will sometimes describe the work in prose instead.
  </Accordion>
  <Accordion title="Python: subagent spans don't nest under the parent">
    The Python integration needs `lmnr-claude-code-proxy`, which is installed automatically as a core dependency of `lmnr`. If you pinned a very old `lmnr` version, upgrade to `lmnr>=0.7.47`. The proxy intercepts the Agent SDK's subprocess calls to the model API and injects trace context so subagent spans nest under the parent run.
  </Accordion>
  <Accordion title="Self-hosting Laminar">
    Export `LMNR_BASE_URL` and `LMNR_GRPC_PORT` alongside `LMNR_PROJECT_API_KEY`. For a local OSS instance, that's `LMNR_BASE_URL=http://localhost` and `LMNR_GRPC_PORT=8001`.
  </Accordion>
</AccordionGroup>

## Overview

Claude Agent SDK lets you call Claude Code from your own code: it manages the underlying process and the model calls so you can drive agents programmatically. Laminar instruments the SDK at the client-method level, so a single `Laminar.initialize()` captures every `query`, `receive_messages`, LLM call, tool call, and (crucially) every subagent the main agent spawns via the `Task` tool.

The payoff is the **transcript view**: Laminar extracts the agent's inputs (from the system and user messages), groups each subagent spawn into a collapsible card that shows its input and final output, and lays everything out as a conversation instead of a span tree.

## What Laminar captures

- The full shape of the SDK call graph: `connect`, `query`, `receive_messages`, `receive_response`, `interrupt`, `disconnect`.
- Every LLM prompt, response, token count, and cost.
- Every tool call (`Bash`, `Read`, `Grep`, `Edit`, etc.) with arguments and results.
- Every subagent spawned via `Task`, grouped into a collapsible card with the subagent's input and final output surfaced at the top.
- A transcript timeline you can scrub with any attached browser session or media.

<Step title="Initialize Laminar">

Call `Laminar.initialize()` once at process start. Wrap the entrypoint with `@observe()` so the whole run groups under a single trace.

```python
import asyncio

from claude_agent_sdk import ClaudeSDKClient
from lmnr import Laminar, observe

Laminar.initialize()

@observe()
async def main():
    async with ClaudeSDKClient() as client:
        await client.query(
            "Explain, with examples, how memoization speeds up recursive "
            "function calls. Use the Fibonacci sequence."
        )
        async for msg in client.receive_response():
            print(msg)

if __name__ == "__main__":
    asyncio.run(main())
```

```typescript {2,5,8}
import { query as origQuery } from '@anthropic-ai/claude-agent-sdk';
import { Laminar } from '@lmnr-ai/lmnr';

Laminar.initialize();

const query = Laminar.wrapClaudeAgentQuery(origQuery);

const run = async () => {
  const result = await query({
    prompt: 'Scan the current directory for TODOs and write a summary markdown file.',
  });
  console.log(result);
};
run();
```

## Transcript view: read the conversation, not the span tree

Open the trace in Laminar and the default view is a transcript. Laminar pulls the agent's inputs from the system and user messages, shows the orchestrator's LLM turns inline, and renders every subagent spawned via `Task` as its own collapsible card with **Input** and **Output** previews at the top.

A span tree tells you the shape of the call graph; a transcript tells you what the agent actually did. For long-running, multi-step agents the difference is the point.

## Subagents: collapsible cards with full context

When the orchestrator uses the `Task` tool to spawn a subagent, Laminar detects the boundary (matching the spawn prompt against the subagent's first LLM call) and groups every span inside that subagent under one card. The card is collapsed by default so you can scan the run end-to-end; expand one to see the subagent's inner LLM turns and tool calls without losing your place.

Try this: spawn a handful of parallel subagents and open the trace.

```python example.py
"""Orchestrator that delegates to 4 parallel research subagents."""
import anyio

from claude_agent_sdk import ClaudeAgentOptions, ClaudeSDKClient
from lmnr import Laminar, observe

Laminar.initialize()

SYSTEM_PROMPT = """You are a lead research orchestrator.

You have access to the Task tool. For every discrete research subtask,
spawn a subagent with Task rather than doing the work inline. Good
candidates for a subagent: reading a specific file, running a web
search, summarizing a section, comparing two items.

After all subagents finish, produce a final consolidated answer."""

USER_PROMPT = """Write a short briefing on Python's asyncio.

Spawn separate Task subagents for each of these, in parallel when
possible:
1. Explain the difference between asyncio.gather and asyncio.wait.
2. Explain structured concurrency with asyncio.TaskGroup (3.11+).
3. Give a short example of cancellation with CancelScope / anyio.
4. List three common pitfalls beginners hit with asyncio.

When every subagent has returned, combine their outputs into a
single briefing with one short section per topic."""


@observe(name="asyncio_research_briefing")
async def run() -> None:
    options = ClaudeAgentOptions(
        system_prompt=SYSTEM_PROMPT,
        allowed_tools=["Task"],
        max_turns=25,
    )
    async with ClaudeSDKClient(options=options) as client:
        await client.query(USER_PROMPT)
        async for message in client.receive_response():
            print(message)


if __name__ == "__main__":
    anyio.run(run)
```

In the trace, each of the four `Task` spawns becomes its own card labelled with the subagent's role. Expand a card to see its LLM turns, tool calls (`Bash`, `Read`, `Grep`, `Edit`), and nested work.

</Note>

## Track outcomes with Signals

Traces tell you what happened on one run. [Signals](/signals/introduction) turn traces into structured outcomes: describe a behavior or failure in natural language ("the orchestrator returned without spawning any subagents", "a subagent Bash call failed and the parent didn't retry") and Laminar extracts matching events across your history and every new trace. Route them to alerts, dashboards, or datasets.

## Query across traces

- **[SQL editor](/platform/sql-editor)** for ad-hoc queries across traces, spans, signals, and evals.
- **SQL API** for programmatic access from scripts and pipelines.
- **[MCP server](/platform/mcp)** to query Laminar from Claude Code, Cursor, or Codex. Useful when you want the agent debugging the trace to read it directly.

## Advanced

- Wrap your own orchestration code with [`observe`](/tracing/structure/observe-decorator) to group a multi-query run under one parent span and attach sessions, user IDs, metadata, or tags.
- Add [tags](/tracing/structure/tags) to a span to slice traces in the UI and in SQL.
- If you also call providers directly (e.g. `anthropic.Anthropic()` or OpenAI), those calls are auto-instrumented once `Laminar.initialize()` has run. See the [Anthropic](/tracing/integrations/anthropic) and [OpenAI](/tracing/integrations/openai) integration pages.
